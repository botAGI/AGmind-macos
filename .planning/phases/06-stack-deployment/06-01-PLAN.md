**lib/compose.sh** (~50 lines):

  - Idempotent: run `(cd "$AGMIND_DIR" && docker compose ps --status running -q 2>/dev/null | wc -l | tr -d ' ') || true` and skip if > 0 with log_info

**lib/health.sh** (~80 lines):
- File header: "AGMind container and Ollama health verification, Sourced by install.sh for Phase 7 (Health)"
- Depends on: lib/common.sh (log_info, die, AGMIND_DIR)
- Internal function `_wait_for_container_health()`:
  - Args: `$1` = service name, `$2` = max_attempts (default 24)
  - Get container ID via `(cd "$AGMIND_DIR" && docker compose ps -q "$1" 2>/dev/null) || true`
  - If empty container ID, die with "Service $1 not found"
  - Counter-based loop (5s intervals, `$2` attempts):
    - `docker inspect --format '{{.State.Health.Status}}' "$container_id" 2>/dev/null || true`
    - If "healthy", log_info "[PASS] ${1} is healthy" and return 0
    - Increment counter, sleep 5
  - On exhaustion: `die "${1} failed health check after $((max_attempts * 5))s" "Check: docker logs \$(cd ${AGMIND_DIR} && docker compose ps -q ${1})"`
- Internal function `_wait_for_running()`:
  - Args: `$1` = service name, `$2` = max_attempts (default 12)
  - Same pattern but checks `docker inspect --format '{{.State.Status}}'` for "running"
  - Used for containers without healthcheck definitions
- Internal function `_check_ollama_health()`:
  - Reuse the wait_for_ollama pattern: counter-based poll of `curl -sf http://localhost:11434/api/tags`
  - 30 attempts x 2s = 60s (same as wait_for_ollama in ollama.sh)
  - log_info "[PASS] Ollama API is healthy" on success
  - die "Ollama API not responding" on failure
- Exported function `phase_7_health()`:
  - Determine which services have healthchecks: `api db_postgres redis`
  - Check COMPOSE_PROFILES from .env to conditionally add `weaviate` (grep for "weaviate" in COMPOSE_PROFILES)
  - Call `_wait_for_container_health` for each healthcheck service
  - Determine remaining services to check for running state. Get all running service names via `(cd "$AGMIND_DIR" && docker compose ps --services 2>/dev/null) || true`, filter out the healthcheck services, call `_wait_for_running` for each remaining one
  - Call `_check_ollama_health`
  - log_info "All services healthy"

**lib/models.sh** (~45 lines):
- File header: "AGMind Ollama model management, Sourced by install.sh for Phase 8 (Models)"
- Depends on: lib/common.sh (log_info, die, LOG_FILE)
- Internal function `_pull_model_if_needed()`:
  - Arg: `$1` = model name
  - Capture model list: `model_list=$(ollama list 2>/dev/null) || true`
  - SIGPIPE-safe check: `if echo "$model_list" | grep -q "$1"; then log_info "Model ${1} already present -- skipping pull"; return 0; fi`
  - log_info "Pulling model: ${1} (this may take a while)..."
  - Run `ollama pull "$1" 2>&1 | tee -a "$LOG_FILE"` -- stdout passthrough for progress
  - log_info "Model ${1} pulled successfully"
  - On failure (set -e will catch): die "Failed to pull model ${1}" "Check network connectivity and model name"
- Exported function `phase_8_models()`:
  - Load model names: `WIZARD_LLM_MODEL` and `WIZARD_EMBED_MODEL` (these are exported globals from wizard.sh, available at runtime)
  - Call `_pull_model_if_needed "$WIZARD_LLM_MODEL"`
  - Call `_pull_model_if_needed "$WIZARD_EMBED_MODEL"`
  - log_info "All models ready"

**lib/openwebui.sh** (~70 lines):
- File header: "AGMind Open WebUI admin initialization, Sourced by install.sh for Phase 6 and Phase 9"
- Depends on: lib/common.sh (log_info, log_warn, AGMIND_DIR), lib/config.sh (_generate_secret)
- Internal function `_inject_admin_credentials()`:
  - Called BEFORE docker compose up (by phase_6_start)
  - env_file="${AGMIND_DIR}/.env", creds_file="${AGMIND_DIR}/credentials.txt"
  - If `grep -q "WEBUI_ADMIN_EMAIL" "$env_file" 2>/dev/null`, skip with log_info (idempotent)
  - Generate admin password: `admin_pass=$(_generate_secret)`
  - Set admin_email="admin@agmind.local"
  - Append to .env: `printf "\n# Open WebUI Admin (auto-created on first startup)\nWEBUI_ADMIN_EMAIL=%s\nWEBUI_ADMIN_PASSWORD=%s\nWEBUI_ADMIN_NAME=AGMind Admin\n" "$admin_email" "$admin_pass" >> "$env_file"`
  - Append to credentials.txt: `printf "\n# Open WebUI Admin\nWEBUI_ADMIN_EMAIL=%s\nWEBUI_ADMIN_PASSWORD=%s\n" "$admin_email" "$admin_pass" >> "$creds_file"`
  - log_info "Open WebUI admin credentials injected into .env"
- Internal function `_verify_openwebui_admin()`:
  - Called by phase_9_complete to verify the admin was created
  - **Step 1 -- Accessibility check (primary):** Counter-based poll: 10 attempts x 3s, `curl -sf http://localhost/ >/dev/null 2>&1`
  - On poll success: log_info "Open WebUI is accessible at http://localhost/"
  - On poll failure: log_warn "Open WebUI not responding at http://localhost/ -- check nginx and open-webui containers". Use log_warn not die (non-fatal per CONTEXT.md). Return early -- skip POST fallback.
  - **Step 2 -- POST signup fallback verification (per locked CONTEXT.md decision):** After accessibility confirmed, read admin_email and admin_pass from credentials.txt (grep WEBUI_ADMIN_EMAIL and WEBUI_ADMIN_PASSWORD, cut -d= -f2). Attempt POST to `http://localhost:3000/api/v1/auths/signup` with JSON payload `{"email":"$admin_email","password":"$admin_pass","name":"AGMind Admin"}`. Use `curl -sf -X POST -H "Content-Type: application/json" -d "$payload" http://localhost:3000/api/v1/auths/signup 2>/dev/null || true`.
  - If curl returns 0 (success): log_info "Admin account created via signup API"
  - If curl returns non-zero (400 = already exists, or other error): log_info "Admin account already exists (signup API returned non-success -- expected for idempotent re-run)"
  - This is non-fatal in all cases. The env var injection is the primary mechanism; POST signup is a belt-and-suspenders fallback that satisfies the locked CONTEXT.md decision.
