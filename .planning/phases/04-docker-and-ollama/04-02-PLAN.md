Create `lib/ollama.sh` with the following structure. Module header must include `set -eEuo pipefail` and a comment block listing all exported functions and dependencies.

**install_ollama():**
- If `OLLAMA_RUNNING` is 1 (set by `detect_ollama()` in Phase 1 diagnostics), log_info "Ollama already running -- skipping install" and return 0
- Check if ollama is installed: `brew list --formula ollama >/dev/null 2>&1` -- if yes, log_info "Ollama already installed" and return 0
- Run `brew install ollama`
- Log: `log_info "Ollama installed via Homebrew"`

**start_ollama():**
- If `OLLAMA_RUNNING` is 1, log_info "Ollama already running on port 11434 -- skipping start" and return 0
- Run `brew services restart ollama` (using restart instead of start for robustness per Research Pitfall 6 -- handles case where plist is loaded but process crashed)
- Log: `log_info "Ollama service started"`
- Call `wait_for_ollama` to confirm readiness

**wait_for_ollama():**
- Counter-based polling loop (NO GNU `timeout` command)
- `local max_attempts=30` and `local attempt=0`
- Log: `log_info "Waiting for Ollama API..."`
- While loop: `while [ "$attempt" -lt "$max_attempts" ]; do`
  - Test: `if curl -sf http://localhost:11434/api/tags >/dev/null 2>&1; then`
  - On success: `log_info "Ollama is ready"` and return 0
  - Increment: `attempt=$((attempt + 1))`
  - Sleep: `sleep 2`
- On timeout: `die "Ollama failed to start within 60 seconds" "Check: brew services list | grep ollama"`

Important constraints:
- Ollama is NEVER in Docker on macOS (add a comment at the top of the file stating this)
- No `timeout` command (Bash 3.2 compatible polling loop only)
- All functions use check-then-skip idempotency pattern
- Use `${OLLAMA_RUNNING:-0}` pattern to handle unset variable under `set -u`
