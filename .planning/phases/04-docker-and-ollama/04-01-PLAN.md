Create `lib/docker.sh` with the following functions. The module header must include `set -eEuo pipefail` and a comment block listing all exported functions. All functions use `log_info`, `log_warn`, `log_debug`, `die` from `lib/common.sh` and read globals from `lib/detect.sh`.

**detect_docker_runtime():**
- If `DOCKER_RUNTIME` env var is already set and non-empty, log it with `log_debug` and return 0 (env override per DOCKER-06)
- Otherwise test Docker Desktop socket at `${HOME}/.docker/run/docker.sock` using `DOCKER_HOST="unix://path" docker info >/dev/null 2>&1`
- Then test Colima socket at `${HOME}/.colima/default/docker.sock` using same pattern
- Prefer Desktop if both respond
- Set `DOCKER_RUNTIME` to "desktop", "colima", or "none" and export it
- Log result with `log_info`

**install_colima():**
- Check each of `colima`, `docker`, `docker-compose` individually with `brew list --formula <pkg> >/dev/null 2>&1`
- Build space-separated string of packages not yet installed
- If string is non-empty, run `brew install $packages_needed` (intentionally unquoted for word splitting)
- Log each already-installed package with `log_info "<pkg> already installed"`
- Log the install command with `log_info "Installing: <packages>"`
- After install, call `setup_compose_plugin` to configure Docker CLI plugin path

**setup_compose_plugin():**
- Create `~/.docker` directory if missing (`mkdir -p`)
- Determine plugins dir: `${BREW_PREFIX}/lib/docker/cli-plugins`
- If `~/.docker/config.json` does not exist, create it with content: `{"cliPluginsExtraDirs": ["<plugins_dir>"]}`
- If file exists but does not contain `cliPluginsExtraDirs`, use `python3 -c` to safely merge the key into existing JSON (python3 is available on stock macOS)
- If already present, log_debug and return
- Use `printf` to write new files (not echo, for portability)

**start_colima():**
- Check if already running: `colima status >/dev/null 2>&1` -- if so, `log_info "Colima already running -- skipping start"` and return 0
- Map `DETECTED_ARCH`: "arm64" -> "aarch64", "x86_64" -> "x86_64", anything else -> `die`
- Read resource overrides: `cpu="${COLIMA_CPU:-8}"`, `memory="${COLIMA_MEMORY:-16}"`, `disk="${COLIMA_DISK:-60}"`
- Log: `log_info "Starting Colima (arch=${arch_flag}, cpu=${cpu}, memory=${memory}GB, disk=${disk}GB)..."`
- Run: `colima start --arch "$arch_flag" --cpu "$cpu" --memory "$memory" --disk "$disk"`
- After start, wait for Docker readiness using a counter-based polling loop: 30 attempts, 2s sleep, test with `docker info >/dev/null 2>&1`, die on timeout with hint "Check: colima status"

**fix_docker_socket():**
- Internal helper `_get_socket_path()`: case on `DOCKER_RUNTIME` -- "desktop" returns `${HOME}/.docker/run/docker.sock`, "colima" returns `${HOME}/.colima/default/docker.sock`, anything else dies
- Get active socket path via `_get_socket_path`
- Verify active socket exists with `[ -S "$active_socket" ]` -- die if not
- If `DOCKER_RUNTIME=desktop` and `/var/run/docker.sock` exists as a socket (not symlink) and `docker info` succeeds without DOCKER_HOST override, log_info "Docker Desktop manages socket -- skipping symlink" and return 0
- If `/var/run/docker.sock` is a symlink pointing to the correct target, log_debug and return 0
- Otherwise: `log_info` the symlink creation, run `sudo ln -sf "$active_socket" /var/run/docker.sock`

**verify_compose():**
- Capture output: `compose_output=$(docker compose version 2>&1)` -- if command succeeds AND output matches `grep -qE 'v[2-9]\.'`, log_info the version and return 0
- Otherwise: `die "Docker Compose v2 not available" "Run: brew install docker-compose && see ~/.docker/config.json cliPluginsExtraDirs"`
