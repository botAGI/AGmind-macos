Create `lib/config.sh` with the following structure. Must be Bash 3.2 compatible (no associative arrays, no mapfile, no ${var,,}). Use `set -eEuo pipefail`. Follow the established module pattern from lib/docker.sh and lib/ollama.sh.

**Header:**
```bash
#!/bin/bash
# lib/config.sh -- AGMind configuration file generation
# Sourced by install.sh for Phase 5 (Configuration)
# Compatible with /bin/bash 3.2.57 (stock macOS)
#
# Provides: Template rendering, secret generation, config assembly
# Exports: phase_5_configuration
#
# Internal: _generate_secret, _load_or_generate_secrets, _write_credentials_file,
#           _render_env_file, _render_nginx_conf, _render_compose_file,
#           _build_compose_profiles, _get_docker_socket_volume
#
# Depends on: lib/common.sh (_atomic_sed, die, log_info, ensure_directory, AGMIND_DIR)
#             lib/wizard.sh (WIZARD_* exports)
#             lib/docker.sh (DOCKER_RUNTIME)
```

**Functions to implement:**

1. `_generate_secret()` -- Generate a 32-char alphanumeric secret:
   `LC_ALL=C tr -dc 'A-Za-z0-9' < /dev/urandom | head -c 32`
   No arguments, echoes the secret string.

2. `_load_or_generate_secrets()` -- Idempotent secret management:
   - If `${AGMIND_DIR}/credentials.txt` exists, read KEY=VALUE pairs from it (skip comments starting with `#` and blank lines) and export each variable.
   - If credentials.txt does NOT exist, generate 9 secrets: DB_PASSWORD, REDIS_PASSWORD, DIFY_SECRET_KEY, OPENWEBUI_SECRET, SANDBOX_API_KEY, WEAVIATE_API_KEY, QDRANT_API_KEY, PLUGIN_DAEMON_KEY, DIFY_INNER_API_KEY. Export all 9. Then call `_write_credentials_file`.
   - Use `while IFS='=' read -r key value` loop. For the case pattern to skip comments, use `\#*|"")` to handle both `#` lines and blank lines.

3. `_write_credentials_file()` -- Write credentials.txt:
   - Path: `${AGMIND_DIR}/credentials.txt`
   - Header: `# AGMind Credentials - Generated <date>`
   - Header: `# Keep this file safe. chmod 600.`
   - One `KEY=VALUE` per line for all 9 secrets
   - Include service URLs for reference: Dify console URL, Open WebUI URL, Ollama API URL
   - Use `sudo tee` to write (directory is owned by root initially), then `sudo chown "$(whoami)"` and `chmod 600`

4. `_build_compose_profiles()` -- Build COMPOSE_PROFILES string:
   - Always starts with `postgresql`
   - Appends `${WIZARD_VECTOR_DB}` (weaviate or qdrant)
   - If `WIZARD_ETL_MODE=extended`, appends `unstructured`
   - If `WIZARD_MONITORING_MODE=local`, appends `monitoring`
   - Echoes comma-separated string

5. `_get_docker_socket_volume()` -- Get Docker socket path for compose volume mount:
   - `desktop` -> `${HOME}/.docker/run/docker.sock`
   - `colima` -> `${HOME}/.colima/default/docker.sock`
   - `*` -> `/var/run/docker.sock` (fallback)

6. `_render_env_file()` -- Render .env from profile template:
   - Determine template: `${SCRIPT_DIR}/templates/env.${WIZARD_DEPLOY_PROFILE}.template`
   - Die if template not found
   - Copy template to `${AGMIND_DIR}/.env` using `sudo cp`, then `sudo chown "$(whoami)"`
   - Substitute all 9 secret placeholders using `_atomic_sed "s|{{PLACEHOLDER}}|${VALUE}|g"` (pipe delimiter to avoid URL slash conflicts)
   - Substitute wizard choices: `{{WIZARD_LLM_MODEL}}`, `{{WIZARD_EMBED_MODEL}}`, `{{WIZARD_VECTOR_DB}}`
   - Build COMPOSE_PROFILES via `_build_compose_profiles` and substitute `{{COMPOSE_PROFILES}}`
   - Determine ETL_TYPE: if `WIZARD_ETL_MODE=extended` then `unstructured`, else `dify`. Substitute `{{ETL_TYPE}}`
   - Log completion

7. `_render_nginx_conf()` -- Render nginx.conf:
   - Copy `${SCRIPT_DIR}/templates/nginx.conf.template` to `${AGMIND_DIR}/nginx.conf`
   - Use `sudo cp` then `sudo chown "$(whoami)"`
   - No substitutions needed for v1 (all values are static Docker service names)
   - Log completion

8. `_render_compose_file()` -- Render docker-compose.yml:
   - Copy `${SCRIPT_DIR}/templates/docker-compose.yml` to `${AGMIND_DIR}/docker-compose.yml`
   - Use `sudo cp` then `sudo chown "$(whoami)"`
   - Get socket path via `_get_docker_socket_volume` and substitute `{{DOCKER_SOCKET_PATH}}`
   - Log completion

9. `phase_5_configuration()` -- Main phase entry point:
   - `log_info "Generating configuration files..."`
   - Call `_load_or_generate_secrets`
   - Call `_render_env_file`
   - Call `_render_nginx_conf`
   - Call `_render_compose_file`
   - Copy `${SCRIPT_DIR}/templates/versions.env` to `${AGMIND_DIR}/versions.env` (sudo cp + chown)
   - `log_info "Configuration complete -- files written to ${AGMIND_DIR}"`

Important implementation notes:
- ALL sed patterns MUST use `|` as delimiter, not `/`, because values contain URLs with slashes
- SCRIPT_DIR is already set by install.sh before lib/config.sh is sourced, so use it directly
- Use `${AGMIND_DIR}` (from common.sh) not hardcoded `/opt/agmind`
- All file writes use `sudo cp` followed by `sudo chown "$(whoami)"` because /opt/agmind may be root-owned
- The credentials.txt read loop must handle the case where value itself contains `=` signs (use `read -r key value` where value gets the rest of the line after first `=`)
