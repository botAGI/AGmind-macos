**LLM model data (pipe-delimited indexed array):**
```bash
_LLM_MODELS=(
    "gemma3:4b|8"
    "qwen2.5:3b|8"
    "qwen2.5:7b|16"
    "llama3.1:8b|16"
    "qwen2.5:14b|32"
    "phi-4:14b|32"
    "gemma3:27b|64"
    "qwen2.5:32b|64"
    "qwen2.5:72b|96"
    "llama3.1:70b|96"
)
```
Use `${entry%%|*}` for model tag, `${entry##*|}` for required RAM. Note pipe delimiter (NOT colon) because model tags contain colons.

**Embed model list:**
```bash
_EMBED_MODELS=("nomic-embed-text" "bge-m3" "mxbai-embed-large")
```

**_get_recommended_model():** Takes RAM in GB as $1. Walks _LLM_MODELS array, finds highest-tier model where required_ram <= detected_ram. Returns the first model tag at that tier. For RAM >= 96, returns qwen2.5:72b. For RAM >= 64, returns gemma3:27b. For RAM >= 32, returns qwen2.5:14b. For RAM >= 16, returns qwen2.5:7b. For RAM >= 8, returns gemma3:4b. Uses echo to return value.

**_validate_choice():** Takes name ($1), value ($2), then valid options ($3...). Loops through valid options checking equality. Returns 0 on match, calls `die "Invalid ${name}: '${value}'" "Valid options: $*"` on no match.

**_wizard_ask():** Generic numbered menu. Takes prompt ($1), default_num ($2), then options ($3...). Uses `printf` to display numbered options with "(default)" on default_num. Reads with `read -r`. Empty input selects default. Validates input is integer 1..count using `case` for numeric check then range check with `2>/dev/null`. Re-prompts "Invalid choice. Please enter 1-N:" on bad input. Returns selected option value via echo.

**_wizard_ask_llm_model():** Special model menu function. Takes DETECTED_RAM_GB. Calls _get_recommended_model() to find recommended index. Displays each model as:
- `[N] model:tag` if within RAM
- `[N] model:tag * (recommended for your XGB)` for the recommended model
- `[N] model:tag (requires NGB+, you have XGB)` if model RAM > system RAM
Default is recommended model number (1-indexed). Uses `read -r`, validates, returns model tag.

**Seven ask functions (or use _wizard_ask for the simple ones):**
1. `_wizard_ask_profile()` -- options: lan, offline. Default: 1 (lan). Explanation: "lan = Local network access, no TLS / offline = Air-gapped, no internet required"
2. `_wizard_ask_llm_model()` -- special model menu (described above)
3. `_wizard_ask_embed_model()` -- options from _EMBED_MODELS. Default: 1 (nomic-embed-text).
4. `_wizard_ask_vector_db()` -- options: weaviate, qdrant. Default: 1 (weaviate).
5. `_wizard_ask_etl_mode()` -- options: standard, extended. Default: 1 (standard).
6. `_wizard_ask_monitoring()` -- options: none, local. Default: 1 (none).
7. `_wizard_ask_backup()` -- options: local, remote. Default: 1 (local).

Each question must show a brief explanation of what the choice affects (per locked decision).

**_wizard_non_interactive():** Reads all 7 values from env vars with defaults:
- `WIZARD_DEPLOY_PROFILE="${DEPLOY_PROFILE:-lan}"` then `_validate_choice`
- `WIZARD_LLM_MODEL="${LLM_MODEL:-$(_get_recommended_model "$DETECTED_RAM_GB")}"` -- NO validation (any model tag accepted per CONTEXT.md)
- `WIZARD_EMBED_MODEL="${EMBED_MODEL:-nomic-embed-text}"` then `_validate_choice`
- `WIZARD_VECTOR_DB="${VECTOR_DB:-weaviate}"` then `_validate_choice`
- `WIZARD_ETL_MODE="${ETL_MODE:-standard}"` then `_validate_choice`
- `WIZARD_MONITORING_MODE="${MONITORING_MODE:-none}"` then `_validate_choice`
- `WIZARD_BACKUP_MODE="${BACKUP_MODE:-local}"` then `_validate_choice`
Exports all 7 WIZARD_* variables. Logs each choice with log_info.

**run_wizard():** Main entry point.
- If `NON_INTERACTIVE=1`: call `_wizard_non_interactive()`
- Else: call each of the 7 ask functions in order, setting WIZARD_* from return values
- Export all 7 WIZARD_* variables
- Log summary of all choices
