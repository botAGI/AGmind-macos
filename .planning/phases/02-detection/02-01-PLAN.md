Create `lib/detect.sh` with the following structure. The file MUST start with `set -eEuo pipefail` and a header comment matching the Phase 1 pattern.

**Individual detection functions (each self-contained, exports variables, returns 0):**

1. `detect_os()` -- Uses `sw_vers -productVersion` for version, `uname -m` for arch. Exports `DETECTED_OS=macos`, `DETECTED_OS_VERSION` (e.g., "14.2" or "26.3"), `DETECTED_ARCH` (arm64 or x86_64). Calls `log_debug` with results.

2. `detect_ram()` -- Uses `sysctl -n hw.memsize` to get bytes, converts with `$((bytes / 1024 / 1024 / 1024))`. Exports `DETECTED_RAM_GB` as integer. Calls `log_debug` with result.

3. `detect_disk()` -- Uses `df -k /` and `awk 'NR==2 {print $4}'` to get available KB, converts with `$((avail_kb / 1024 / 1024))`. Exports `DETECTED_DISK_FREE_GB` as integer. Calls `log_debug` with result.

4. `_check_port()` -- Internal helper. Takes port number as $1. Runs `lsof -iTCP:"${port}" -sTCP:LISTEN -P -n 2>/dev/null`. If exit code 0 (port in use), extracts process name and PID with `awk 'NR==2 {print $1}'` and `awk 'NR==2 {print $2}'`, echoes `"processname:pid"`, returns 0. If exit code 1 (port free), returns 1. Use `|| return 1` pattern to avoid ERR trap firing.

5. `detect_ports()` -- Checks ports 80, 443, 3000, 11434 using `_check_port`. Builds newline-separated `PORT_CONFLICTS` string (format: `"port:processname:pid"` per line). Does NOT export PORT_CONFLICTS (it is used internally by preflight_checks). Calls `log_debug` for each port checked.

6. `_test_docker_socket()` -- Internal helper. Takes socket path as $1. Checks `[ -S "$socket" ]` first, then `DOCKER_HOST="unix://${socket}" docker info >/dev/null 2>&1`. Returns 0 if connected, 1 if not. Use `|| return 1` on both checks.

7. `detect_docker()` -- First checks for `DOCKER_RUNTIME` env override: if set and non-empty, log it and return 0. Otherwise auto-detects by testing Docker Desktop socket (`"${HOME}/.docker/run/docker.sock"`) then Colima socket (`"${HOME}/.colima/default/docker.sock"`) via `_test_docker_socket`. Prefers Desktop if both respond. Sets `DOCKER_RUNTIME=desktop|colima|none`. Exports `DOCKER_RUNTIME`. Calls `log_debug` with result.

8. `detect_ollama()` -- Uses `_check_port 11434` to see if something is on port 11434, then optionally verifies with `curl -sf http://localhost:11434/api/tags >/dev/null 2>&1`. Sets `OLLAMA_RUNNING=1` if responding, `OLLAMA_RUNNING=0` if not. Exports `OLLAMA_RUNNING`. Calls `log_debug` with result.

9. `detect_homebrew()` -- Uses `command -v brew >/dev/null 2>&1` to check presence. If found: `BREW_INSTALLED=1`, `BREW_PREFIX=$(brew --prefix)`. If not found: `BREW_INSTALLED=0`, set `BREW_PREFIX="/opt/homebrew"` for arm64 or `"/usr/local"` for x86_64 based on `uname -m`. Exports `BREW_INSTALLED` and `BREW_PREFIX`. Calls `log_debug` with results.

**Preflight aggregator (separate from individual detectors):**

10. `preflight_checks()` -- The main orchestrator function. Structure:

First: Check `SKIP_PREFLIGHT` -- if `"${SKIP_PREFLIGHT:-0}" = "1"`, log_warn that preflight is being skipped and return 0.

Then: Initialize counters: `local fail_count=0 warn_count=0`, and newline-separated accumulator strings: `local fail_msgs="" warn_msgs=""`.

Define three internal helper functions:
- `_preflight_pass() { log_info "[PASS] $1"; }`
- `_preflight_warn() { log_warn "[WARN] $1"; warn_msgs="${warn_msgs}${1}\n"; warn_count=$((warn_count + 1)); }` -- NOTE: use actual newline in the string, not literal `\n`. Do this with: `warn_msgs="${warn_msgs}  - ${1}"$'\n'` -- WAIT, `$'\n'` is Bash 3.2 safe. Use printf: `warn_msgs="${warn_msgs}$(printf '\n')  - ${1}"` or just append with a literal newline between double quotes.
- `_preflight_fail() { log_error "[FAIL] $1"; fail_msgs="${fail_msgs}$(printf '\n')  - ${1}"; fail_count=$((fail_count + 1)); }`

Actually for Bash 3.2 safety, use a simple approach: build the message lists as multi-line strings by echoing into them:

```bash
_preflight_fail() {
    log_error "[FAIL] $1"
    fail_msgs="${fail_msgs}
  - $1"
    fail_count=$((fail_count + 1))
}
```

(The embedded literal newline in the string assignment is Bash 3.2 safe.)

Run all detection functions: `detect_os`, `detect_ram`, `detect_disk`, `detect_homebrew`, `detect_docker`, `detect_ollama`.

Then apply threshold checks per CONTEXT.md locked decisions:

- **macOS version check:** Extract major version with `local major_ver="${DETECTED_OS_VERSION%%.*}"`. If `major_ver -lt 13` then `_preflight_fail "macOS ${DETECTED_OS_VERSION} -- minimum macOS 13 required" `. Else `_preflight_pass "macOS ${DETECTED_OS_VERSION}"`.

- **Architecture check:** If `DETECTED_ARCH` is `arm64` or `x86_64`, `_preflight_pass "Architecture: ${DETECTED_ARCH}"`. Else `_preflight_fail "Unsupported architecture: ${DETECTED_ARCH}"`.

- **RAM check:** If `DETECTED_RAM_GB -lt 8` then `_preflight_fail "RAM: ${DETECTED_RAM_GB}GB (minimum 8GB required)"`. Else `_preflight_pass "RAM: ${DETECTED_RAM_GB}GB"`.

- **Disk check:** If `DETECTED_DISK_FREE_GB -lt 30` then `_preflight_fail "Disk: ${DETECTED_DISK_FREE_GB}GB free (minimum 30GB required)"`. Else `_preflight_pass "Disk: ${DETECTED_DISK_FREE_GB}GB free"`.

- **Port checks:** Run `detect_ports` (which sets PORT_CONFLICTS). For each port in 80, 443, 3000, 11434: check if that port appears in PORT_CONFLICTS. Use `echo "$PORT_CONFLICTS" | grep "^${port}:"` to find matches. If port 80 in use: `_preflight_fail "Port 80 in use by ${proc_name} (PID ${pid}) -- nginx needs this port"`. If port 3000 in use: `_preflight_fail "Port 3000 in use by ${proc_name} (PID ${pid}) -- Dify console needs this port"`. If port 443 in use: `_preflight_warn "Port 443 in use by ${proc_name} (PID ${pid}) -- not required for LAN profile"`. If port 11434 in use: `_preflight_warn "Port 11434 in use by ${proc_name} (PID ${pid}) -- Ollama may already be running (OK)"`.

- **Homebrew check:** If `BREW_INSTALLED -eq 0` then `_preflight_warn "Homebrew not found -- will be needed for Ollama and Docker setup"`. Else `_preflight_pass "Homebrew: ${BREW_PREFIX}"`.

- **Docker check:** If `DOCKER_RUNTIME = "none"` then `_preflight_warn "No Docker runtime found -- will be installed in Phase 3"`. Else `_preflight_pass "Docker: ${DOCKER_RUNTIME}"`.

**Summary section:**

Print a divider line. If `fail_count -gt 0`: print all fail messages, then call `die "Preflight failed with ${fail_count} error(s)" "Fix the issues above and re-run the installer"`.

If `warn_count -gt 0` and `NON_INTERACTIVE != 1`: print all warn messages, then prompt user: `read -r -p "Continue with ${warn_count} warning(s)? [Y/n] " answer`. If answer is "n" or "N", exit 1. If `NON_INTERACTIVE = 1`: log that warnings are being accepted automatically.

If no fails and no warns: `log_info "All preflight checks passed"`.

**IMPORTANT Bash 3.2 constraints:**
- No `declare -A` (associative arrays)
- No `${var,,}` (lowercase transform) -- use case statement or tr
- No `mapfile` or `readarray`
- No `$'\n'` in contexts where it might not work -- use literal newlines in quoted strings
- All functions must handle the `set -e` context by using `|| true` or `if/then` for expected-failure commands
- Use `local` for all function-scoped variables
